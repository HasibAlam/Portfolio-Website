CI/CD Pipeline for Lottery Game

This project demonstrates the implementation of a fully automated CI/CD pipeline for a lottery game built using the .NET framework in C#. The pipeline utilizes GitHub for version control, GitHub Actions for CI/CD automation, and Azure for hosting the instance and site analytics.

Functionality

  • Version control is handled by GitHub, and the CI/CD pipeline is implemented using GitHub Actions workflows.
  • Azure is used to host the application, providing real-time analytics and monitoring.
  • The lottery game, built using the .NET framework in C#, allows users to guess a number between 1 and 10. If the guessed number matches the winning number, the user wins; otherwise, they lose.
  • The pipeline automates the building, testing, and deployment of the application to Azure whenever a new change is pushed to the GitHub repository.
  • The pipeline also integrates NUnit testing to ensure that the core functionality of the lottery game works correctly before deployment.

CI/CD Pipeline Details

The pipeline is built using GitHub Actions, where workflows are defined to automate the building, testing, and deployment processes. The pipeline is fully automated, meaning that any change made to the GitHub repository will trigger the following steps:

  • **Build**: The application is built using .NET commands to ensure that the code compiles and is ready for deployment.
  • **Test**: NUnit tests are executed to verify the functionality of the game logic, particularly the number generation and the win/lose conditions.
  • **Deploy**: The application is deployed to Azure, where it is hosted and monitored.
